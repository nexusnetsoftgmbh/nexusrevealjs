PWA? SPA?

Lifecycle and features

To best understand the workings of a SPA we should compare it to a traditional website.

Regular web applications

Regular web application

  • JS loaded again on every request
  • Time to first byte occurs multiple times
  • Traditional routing (Backend based, direct links to files, etc.)
  • New pages are loaded completely on each request
  • Whole website is reloading

Single page applications

Single page applications

  • JS is loaded only once
  • Time to first byte occurs only once
  • Routing through JS
  • Content updated via ajax
  • Faster/more efficient loading
  • Smoother experience for user
  • Prominent Example: Gmail

So...

Main differences

  • PWA is mobile first
  • It's designed to behave like a native app
  • It includes a Web App Manifest
  • It includes a Service Worker
  • It requires HTTPS

PWA initial request

PWA initial request

  • Service Worker is between app and network
  • URL is cached on visit if not excluded
  • URL is served through the app

Whats the benefit?

PWA offline request

PWA offline request

  • Service Worker checks for cached pages
  • Page is served from cache, if it's been cached
  • If the page has not yet been cached, a custom offline content can be displayed
  • Can even be used to continue shopping if you're offline e.g. in Vue Storefront thanks to background synchronization

Feature overview

  • Fast and efficient
  • Behaves like a native (mobile) app
  • Supports offline features using a service worker
  • Can be added to home screen thanks to a web app manifest
  • Supports Push notifications
  • Doesn't require installation from user

Conclusion

  • A SPA can be a PWA, but a PWA does not have to be a SPA
  • Progressive Web Apps are the next level of browser based applications, because they:
    • Offer rich user experience that parallels thos of native apps
    • Combine features from native apps with the benefits of browser based applications

They are not the same, because...

  • PWAs need to register a service worker, including a fetch event handler and (minimal) offline support
  • PWAs need to have a valid Web Manifest file with a minimal set of icons
  • PWAs need to always be served via HTTPS
